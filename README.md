## 📌 What Is ReceiptMind?

ReceiptMind is a full-stack web app that turns a receipt photo into structured expense data in seconds. Upload a photo of any receipt — grocery store, restaurant, gas station, pharmacy — and Gemma 4 reads it like a human would and returns clean, structured data automatically saved to your personal expense dashboard.

No manual entry. No OCR preprocessing. No template matching. Just AI vision.

Built as a submission for the [Google Gemma 4 Challenge on DEV.to](https://dev.to/challenges/google-gemma-2026-05-06) and as a core feature of a larger personal finance app in development.

---

## ✨ Features

- 📷 **Upload any receipt photo** — JPG or PNG from your phone or desktop
- 🤖 **Gemma 4 reads the image** — extracts merchant, total, date, tax, and category
- 🗂️ **Auto-categorises** — Food & Dining, Groceries, Transport, Healthcare, Entertainment
- 📊 **Live expense dashboard** — running total and full receipt history
- 🗄️ **Persistent storage** — all receipts saved to Neon Postgres
- ⚡ **Zero manual entry** — from upload to dashboard in under 30 seconds

---

## 🏗️ Architecture

```
User uploads receipt image
        ↓
Express backend (multer) receives the file
        ↓
Image converted to base64
        ↓
OpenRouter → Gemma 4 26B MoE (multimodal vision)
        ↓
Gemma returns structured JSON
        ↓
JSON saved to Neon Postgres
        ↓
Dashboard updates with new receipt + running totals
```

## 🛠️ Tech Stack

| Layer | Technology |
|---|---|
| Frontend | HTML + Vanilla JavaScript |
| Backend | Node.js + Express |
| AI Model | Gemma 4 26B MoE via OpenRouter |
| Database | Neon Postgres (serverless) |
| File Handling | Multer (in-memory buffer → base64) |

## 🚀 Getting Started

### Prerequisites

- Node.js 18 or higher
- A free [OpenRouter](https://openrouter.ai) account
- A free [Neon](https://neon.tech) database

### 1. Clone the repo

```bash
git clone https://github.com/Hema-Nambi/ReceiptMind.git
cd ReceiptMind
```

### 2. Install dependencies

```bash
npm install
```

### 3. Set up environment variables

Create a `.env` file in the root folder:

```env
OPENROUTER_API_KEY=sk-or-v1-your-key-here
DATABASE_URL=postgresql://your-neon-connection-string
```

**Get your OpenRouter key:**
1. Go to [openrouter.ai](https://openrouter.ai) → Sign up free
2. Dashboard → API Keys → Create Key
3. The model used is `google/gemma-4-26b-a4b-it:free` — no credits needed

**Get your Neon connection string:**
1. Go to [neon.tech](https://neon.tech) → Sign up free
2. Create project → Dashboard → copy the connection string

### 4. Run the app

```bash
node server.js
```

Open your browser at **http://localhost:3000**

The database table is created automatically on first run.
